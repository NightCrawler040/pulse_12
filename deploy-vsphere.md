# 🐧 Руководство по развертыванию Pulse 12 на Ubuntu Linux (vSphere VM) с PostgreSQL

Данное руководство поможет вам развернуть корпоративный трекер задач **Pulse 12** с выделенной реляционной базой данных **PostgreSQL 16** на вашей виртуальной машине **Ubuntu Linux 22.04 / 24.04** в облаке **vSphere**.

---

## 🛠 Вариант 1: Запуск через Docker Compose (Рекомендуемый для PostgreSQL)

В проекте настроена связка из двух контейнеров: базы данных **PostgreSQL 16** и приложения **Pulse12 Node.js / React**. Развертывание займет 1 минуту:

1. **Установите Docker (если не установлен):**
   ```bash
   curl -fsSL https://get.docker.com -o get-docker.sh
   sudo sh get-docker.sh
   sudo apt-get install -y docker-compose-plugin
   ```

2. **Скопируйте проект на сервер:**
   Скопируйте репозиторий на вашу виртуальную машину (например, в директорию `/opt/pulse12`).

3. **Запустите контейнеры в фоновом режиме:**
   Перейдите в директорию проекта и выполните:
   ```bash
   cd /opt/pulse12
   sudo docker compose up --build -d
   ```

4. **Готово!**
   Сервер запустится на 80-м порту. При первом запуске PostgreSQL автоматически создаст таблицы и заполнит базу данными сотрудников компании (12 учетных записей, демо-спринты, задачи и группы). Все сотрудники в офисе могут открыть в браузере IP вашей машины: `http://IP_АДРЕС_ВАШЕГО_СЕРВЕРА/`.

---

## 🐘 Администрирование базы данных PostgreSQL и Резервное копирование

База данных надежно хранится в изолированном Docker-томе `pulse12_postgres_data`. Для управления используйте стандартные команды:

### 💾 Создание резервной копии (Backup)
Выполните эту команду на вашем сервере Ubuntu для создания дампа базы в файл:
```bash
sudo docker exec -t pulse12-postgres pg_dump -U pulse12_admin pulse12 > pulse12_backup_$(date +%F).sql
```

### ♻️ Восстановление из резервной копии (Restore)
Для восстановления данных из ранее созданного `.sql` файла:
```bash
cat pulse12_backup_2026-07-05.sql | sudo docker exec -i pulse12-postgres psql -U pulse12_admin -d pulse12
```

### 🔍 Подключение к консоли PostgreSQL (psql)
Если вам нужно выполнить прямые SQL-запросы в консоли:
```bash
sudo docker exec -it pulse12-postgres psql -U pulse12_admin -d pulse12
# Примеры команд внутри psql:
# \dt                   -- список всех таблиц
# SELECT * FROM pulse_store; -- просмотр сохраненных коллекций
# \q                    -- выход
```

## ⚙️ Вариант 2: Запуск без Docker (Режим Dual-Mode Fallback)

Приложение **Pulse 12** оснащено умным адаптером базы данных (Dual-Mode):
- Если переменная окружения `DATABASE_URL` задана, сервер подключается к **PostgreSQL**.
- Если вы запускаете `node server/index.js` локально без установленного PostgreSQL или Docker, сервер автоматически и бесшовно переключается в режим **локального файлового хранилища (`server/data/db.json`)**, что позволяет продолжать разработку и тестирование без дополнительных настроек на любом компьютере.

## 🔒 Настройка брандмауэра (UFW) в Ubuntu
Не забудьте открыть порт 80 для сотрудников вашей корпоративной сети LAN:
```bash
sudo ufw allow 80/tcp
sudo ufw reload
```
